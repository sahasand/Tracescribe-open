SDTM Engine SPEC-DRIVEN

Recreation of a spec-driven SDTM build engine: a raw EDC export plus one declarative YAML spec per domain produces a CDISC SDTM package (data, mapping spec, Define-XML), judged against SDTMIG 3.4 + CDISC CT, never a vendor template. A deterministic, seeded generator produces the raw EDC itself, then a minimal in-browser interpreter of the engine’s op vocabulary (const / copy / ct / dtc / derive / seq + a wide-to-long builder) maps it, recording the source of every value. Every mapped value traces to a real generated field; required variables with no collected source surface as an honest pending blank, never a guess. Synthetic study, no real subjects.
